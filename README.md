# Сервис задач

HTTP API модуля трекера задач для медицинской информационной системы.

В исходном проекте уже был реализован CRUD обычных задач: создание, редактирование, удаление, получение по `id` и получение списка. Через этот модуль врачи и персонал ставят себе рабочие задачи, например обзвон пациентов, обход или подготовку отчетности.

В рамках тестового задания добавлена поддержка периодических задач и приоритета задач.

## Что реализовано

- сохранен CRUD обычных задач `/api/v1/tasks`
- добавлен CRUD шаблонов периодических задач `/api/v1/recurring-tasks`
- шаблон периодической задачи автоматически создает обычные задачи в трекере
- добавлен приоритет задачи: `normal` и `urgent`
- экстренные задачи попадают в отдельную очередь `urgent`
- экстренные задачи автоматически переводятся в статус `in_progress`
- добавлен встроенный фронт для удобной работы с задачами и шаблонами через браузер
- Swagger и OpenAPI обновлены

## Периодичность

- `daily` — каждый `n`-й день
- `monthly` — каждое число месяца от `1` до `30`
- `specific_dates` — только в указанные даты
- `day_parity` — только в четные или нечетные дни месяца

## Приоритет и очереди

- `normal` — обычная задача, попадает в очередь `regular`
- `urgent` — экстренная задача, попадает в очередь `urgent`
- если задача создается как `urgent`, она сразу получает статус `in_progress`
- список `GET /api/v1/tasks` возвращает две очереди:
  - `urgent_queue`
  - `regular_queue`

Для сгенерированных задач из периодических шаблонов приоритет наследуется от шаблона.

## Допущения

- все вычисления выполняются по UTC-дате
- уже созданные задачи не изменяются при редактировании периодического шаблона
- при удалении шаблона будущая генерация прекращается, уже созданные задачи сохраняются
- для `specific_dates` даты должны быть уникальными и не раньше `start_date`
- для `monthly` используются только числа `1..30`
- если в месяце нужной даты нет, задача в этот месяц не создается

## Запуск

Требования:

- Go `1.23+`
- Docker и Docker Compose

Запуск через Docker:

```bash
docker compose down -v
docker compose up --build
```

После запуска сервис доступен по адресу `http://localhost:8080`.

Встроенный интерфейс:

```text
http://localhost:8080/
```

Через встроенный фронт можно:

- создавать обычные задачи
- создавать периодические задачи
- работать с экстренной и обычной очередью
- удалять задачи и шаблоны
- вводить даты через календарь в браузере

Swagger:

```text
http://localhost:8080/swagger/
```

## Основные маршруты

- `POST /api/v1/tasks`
- `GET /api/v1/tasks`
- `GET /api/v1/tasks/{id}`
- `PUT /api/v1/tasks/{id}`
- `DELETE /api/v1/tasks/{id}`
- `POST /api/v1/recurring-tasks`
- `GET /api/v1/recurring-tasks`
- `GET /api/v1/recurring-tasks/{id}`
- `PUT /api/v1/recurring-tasks/{id}`
- `DELETE /api/v1/recurring-tasks/{id}`
